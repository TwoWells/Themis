# Getting Started

This guide walks you through installing TheMan and creating your first profile.

## Installation

### From Source

Requires [Rust](https://rustup.rs/) 1.70+.

```bash
git clone https://github.com/yourusername/theman.git
cd theman

# User install (no sudo, installs to ~/.local)
make install PREFIX=~/.local

# System install (requires sudo, installs to /usr/local)
sudo make install
```

For user installs, ensure `~/.local/bin` is in your `PATH`.

### Shell Completions

System installs include shell completions automatically. For user installs, add to your shell rc
file:

```bash
# Bash (~/.bashrc)
eval "$(theman completions bash)"

# Zsh (~/.zshrc)
eval "$(theman completions zsh)"

# Fish (~/.config/fish/config.fish)
theman completions fish | source
```

## Initialize Configuration

Create the configuration directory structure:

```bash
theman init
```

This creates:

```
~/.config/theman/
├── theman.yaml           # Main configuration
├── profiles/
│   └── example.yaml      # Sample profile
├── palettes/             # User palettes
└── templates/            # Jinja2 templates
```

## Your First Profile

### 1. Create a Profile

Edit `~/.config/theman/profiles/dark.yaml`:

```yaml
vars:
  bg: "#1a1a2e"
  fg: "#eaeaea"
  accent: "#e94560"
  font_family: "JetBrains Mono"
  font_size: 12
```

### 2. Enroll an Application

Edit `~/.config/theman/theman.yaml` to enroll kitty:

```yaml
enroll:
  kitty:
    type: template
    input: "~/.config/theman/templates/kitty.j2"
    output: "~/.config/kitty/.theman.conf"
```

### 3. Create a Template

Create `~/.config/theman/templates/kitty.j2`:

```jinja2
# Generated by TheMan - do not edit
foreground {{ fg }}
background {{ bg }}
font_family {{ font_family }}
font_size {{ font_size }}
```

### 4. Include the Generated File

Add this line to `~/.config/kitty/kitty.conf`:

```
include .theman.conf
```

### 5. Load the Profile

```bash
# Preview first
theman load dark --dry-run

# Apply for real
theman load dark
```

## Verify Your Setup

Check that everything is configured correctly:

```bash
# Validate YAML syntax and references
theman verify

# Check that apps have the include patterns
theman doctor
```
